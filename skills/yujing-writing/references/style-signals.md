# Style Signals

Use this reference to match Yujing's team-facing style when writing strategy, planning, or operating documents.

## High-Confidence Signals

1. Lead with judgment, then expand.
2. Prefer section labels that are direct nouns, such as `趋势` and `今年的目标`.
3. Use short bullets to express one idea per line.
4. Mix strategic statements with immediate execution implications.
5. Use numbered decomposition for goals and sub-actions.
6. Emphasize leverage: better models, automation, validation, and toolchain fit.
7. Balance AI-first execution with explicit human review and acceptance boundaries.

## Sentence Patterns

1. Contrast pattern: `X is becoming cheap; Y is still expensive.`
2. Inflection pattern: `Quality and efficiency have crossed a threshold.`
3. Directive pattern: `Use the best model.`
4. Operating principle pattern: `If work is manual, check if it can be automated.`
5. Tradeoff pattern: `Editor is fine; CLI is better in high-generation workflows.`
6. System expansion pattern: `Programming is not just coding; it is a universal automation interface.`

## Structure Pattern

Use this default shape for strategy docs:

1. `趋势`:
   - Describe external change.
   - Explain why it matters now.
   - State operating implication.
2. `今年的目标`:
   - List top-level goals in numbered form.
   - Break each goal into numbered execution items.
3. `工程实践`:
   - Specify review model, acceptance boundary, and repository-level conventions.
4. `产品方向`:
   - Explain how to make products easier for AI systems to understand and operate.
5. `下一步`:
   - List owner-facing actions and dependencies.

## Vocabulary Preferences

Prefer:
`模型`, `自动化`, `验证`, `review`, `验收边界`, `工具链`, `CLI`, `OpenAPI`, `Action`, `Scale`, `Friction`.

Avoid:
high-level motivational wording without operating detail.
